# Маркетплейс одежды (Clothing Marketplace)

Веб-приложение, API и мобильное приложение для маркетплейса одежды. Проект объединяет три компонента: веб-сайт для покупателей и продавцов, REST API бекенд и мобильное приложение.

---

## О проекте

Маркетплейс одежды — это платформа, где продавцы размещают товары, а покупатели просматривают каталог, добавляют товары в корзину и оформляют заказы. Проект демонстрирует полноценную e-commerce систему с разделением ролей и типичной бизнес-логикой.

### Компоненты

| Компонент | Описание |
|-----------|----------|
| **Веб-сайт** | Интерфейс для покупателей и продавцов: каталог, корзина, заказы, управление товарами |
| **API (Backend)** | REST API для авторизации, работы с товарами, корзиной и заказами |
| **Мобильное приложение** | Клиент для покупателей: каталог, карточка товара, корзина, заказы |

---

## Роли пользователей

| Роль | Возможности |
|------|-------------|
| **Покупатель** | Просмотр каталога, фильтры, корзина, оформление заказа, история заказов, профиль |
| **Продавец** | Добавление/редактирование товаров, просмотр и обработка заказов |
| **Админ** | Управление пользователями, категориями, модерация товаров |

## Структура данных

### Основные сущности

```
User (Пользователь)
├── id, email, password, name
├── role (buyer / seller / admin)
└── delivery address(es)

Category (Категория)
└── name (Женщинам, Мужчинам, Детям, Аксессуары...)

Product (Товар)
├── name, description, price
├── category_id, seller_id
├── sizes (S, M, L, XL)
├── colors
├── images (массив фото)
└── status (active / moderation)

Cart (Корзина)
├── user_id
└── items (product_id, size, color, quantity)

Order (Заказ)
├── buyer_id, seller_id
├── items, total_amount, status
├── delivery_address
└── created_at
```

### Связи

- **Product** → Category (многие к одному)
- **Product** → User (продавец)
- **Cart** → User (один к одному)
- **Order** → User (покупатель), Order → User (продавец)
- **OrderItem** → Product, OrderItem → Order

## API (структура эндпоинтов)

### Авторизация
```
POST   /api/auth/register      — Регистрация
POST   /api/auth/login         — Вход (получение JWT)
GET    /api/auth/me            — Текущий пользователь
```

### Категории
```
GET    /api/categories         — Список категорий
GET    /api/categories/:id     — Категория по ID
POST   /api/categories         — Создание (admin)
PUT    /api/categories/:id     — Обновление (admin)
DELETE /api/categories/:id     — Удаление (admin)
```

### Товары
```
GET    /api/products/              — Список (фильтры: category, size, color, min_price, max_price, search)
GET    /api/products/:id/          — Товар по ID (публичный)
POST   /api/products/create/       — Создание (seller)
PUT    /api/products/:id/manage/   — Обновление (seller)
DELETE /api/products/:id/manage/   — Удаление (seller)
POST   /api/products/:id/images/   — Загрузка фото
```

### Корзина
```
GET    /api/cart               — Содержимое корзины
POST   /api/cart/items         — Добавить позицию
PUT    /api/cart/items/:id     — Изменить количество
DELETE /api/cart/items/:id     — Удалить позицию
```

### Заказы
```
GET    /api/orders/            — Список заказов пользователя
GET    /api/orders/:id/        — Заказ по ID
POST   /api/orders/create/     — Создание заказа из корзины
PUT    /api/orders/:id/status/ — Изменение статуса (seller/admin)
```

## Установка и запуск

### Backend

1. Перейдите в папку backend:
   ```bash
   cd backend
   ```

2. Создайте виртуальное окружение:
   ```bash
   python -m venv venv
   venv\Scripts\activate   # Windows
   # source venv/bin/activate   # Linux/Mac
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

4. Настройте базу данных (опционально):
   - **PostgreSQL**: создайте БД и скопируйте `.env.example` в `.env`, укажите `DB_NAME`, `DB_USER`, `DB_PASSWORD` и т.д.
   - **SQLite** (по умолчанию): если `DB_NAME` не задан, используется SQLite — ничего настраивать не нужно.

5. Выполните миграции:
   ```bash
   python manage.py migrate
   ```

6. Создайте суперпользователя (для админки):
   ```bash
   python manage.py createsuperuser
   ```

7. Запустите сервер:
   ```bash
   python manage.py runserver
   ```

API будет доступен по адресу: `http://127.0.0.1:8000/api/`  
Админ-панель: `http://127.0.0.1:8000/admin/`

### Frontend (Web)

1. Перейдите в папку frontend:
   ```bash
   cd frontend
   ```

2. Установите зависимости (если ещё не установлены):
   ```bash
   npm install
   ```

3. Запустите dev-сервер:
   ```bash
   npm run dev
   ```

4. Откройте http://localhost:3000

**Важно:** Django backend должен быть запущен на порту 8000 — Vite проксирует `/api` и `/media` на бэкенд.
